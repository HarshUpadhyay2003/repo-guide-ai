# Support

Need help with RepoGuideAI? We are here to assist you!

## Where to Get Help

1. **GitHub Issues**: If you have found a bug, want to request a new feature, or need to report unexpected behavior with the API outputs, please open a GitHub Issue.
2. **GitHub Discussions**: For general questions, architecture discussions, or help setting up your local environment, please utilize the GitHub Discussions board.
3. **Documentation**: The primary documentation is available in the `README.md` file at the root of the project. Make sure to check there first for setup and architecture details!
4. **API Reference**: Once you have the backend running locally, comprehensive interactive API documentation is automatically generated and available via Swagger UI at `http://localhost:8000/docs`.

## How to Report Bugs

When reporting a bug on the issue tracker, please include:
- A descriptive title.
- The steps necessary to reproduce the bug.
- What you expected to happen vs. what actually happened.
- Relevant logs, error messages, or screenshot(s).
- Information about your local environment (OS, Python version).

## Feature Requests
RepoGuideAI is rapidly evolving from an AI repository summarizer to a full Open Source Contribution Assistant. If you have ideas for new features, please submit an issue labeled as `enhancement`. Include a clear description of the use case and how it fits into the broader onboarding and contribution workflow.
